This guide walks you through registering a Medplum project and making your first FHIR API calls using the `@medplum/core` SDK.

<Note>
  Medplum is available as a hosted cloud service at [app.medplum.com](https://app.medplum.com) or as a self-hosted deployment on your own infrastructure. This guide uses the hosted cloud. To self-host, see the [self-hosting docs](/self-hosting/index).
</Note>

<Steps>

  <Step title="Register at app.medplum.com">
    Open [https://app.medplum.com/register](https://app.medplum.com/register) in your browser.

    Fill in your account details and click **Create account**. On the next screen, enter a name for your first **Medplum Project** and click **Create Project**.

    A project is a logically isolated collection of FHIR resources with its own users, permissions, and client applications. A common pattern is to create separate staging and production projects.

    After registration, note your project ID from the project settings page — you will use it when configuring client credentials.
  </Step>

  <Step title="Install the SDK">
    Install `@medplum/core` and the FHIR type definitions:

    <CodeGroup>

    ```bash npm
    npm install @medplum/core @medplum/fhirtypes
    ```

    ```bash yarn
    yarn add @medplum/core @medplum/fhirtypes
    ```

    ```bash pnpm
    pnpm add @medplum/core @medplum/fhirtypes
    ```

    </CodeGroup>

    `@medplum/core` is the main SDK package (version `5.1.6`). It ships both ESM and CJS builds and requires Node.js `^22.18.0` or `>=24.2.0`.

    `@medplum/fhirtypes` provides TypeScript type definitions for all FHIR R4 resources. It has no runtime footprint.
  </Step>

  <Step title="Initialize MedplumClient">
    Create a `MedplumClient` instance. For server-side or CLI usage, pass your `baseUrl` and credentials. For browser apps, `baseUrl` defaults to `https://api.medplum.com/`.

    ```typescript client.ts
    import { MedplumClient } from '@medplum/core';

    const medplum = new MedplumClient({
      baseUrl: 'https://api.medplum.com/',
      clientId: 'YOUR_CLIENT_ID',
      clientSecret: 'YOUR_CLIENT_SECRET',
    });
    ```

    Key `MedplumClientOptions` fields:

    | Option | Default | Description |
    |---|---|---|
    | `baseUrl` | `https://api.medplum.com/` | Base URL of the Medplum server |
    | `clientId` | — | OAuth2 client ID from your project |
    | `clientSecret` | — | OAuth2 client secret (server-side only) |
    | `accessToken` | — | Use a pre-obtained access token directly |
    | `fhirUrlPath` | `fhir/R4/` | Path to the FHIR endpoint |
    | `cacheTime` | `60000` | Resource cache TTL in milliseconds |
    | `logLevel` | `'none'` | `'none'` \| `'basic'` \| `'verbose'` |

    <Tip>
      In a browser app using PKCE, omit `clientSecret` and call `startPkceLogin()` instead. The client will handle token exchange automatically.
    </Tip>
  </Step>

  <Step title="Authenticate with client credentials">
    For server-side or automated scripts, authenticate using OAuth2 client credentials:

    ```typescript auth.ts
    import { MedplumClient } from '@medplum/core';

    const medplum = new MedplumClient({
      baseUrl: 'https://api.medplum.com/',
    });

    // Authenticate with client ID and secret
    await medplum.startClientLogin('YOUR_CLIENT_ID', 'YOUR_CLIENT_SECRET');

    console.log('Authenticated as project:', medplum.getActiveLogin()?.project?.display);
    ```

    After a successful login, the client stores the access token and refreshes it automatically before expiry (5 minutes before expiration by default).
  </Step>

  <Step title="Create a Patient resource">
    Once authenticated, you can create FHIR resources using `createResource`:

    ```typescript create-patient.ts
    import { MedplumClient } from '@medplum/core';
    import type { Patient } from '@medplum/fhirtypes';

    const medplum = new MedplumClient({ baseUrl: 'https://api.medplum.com/' });
    await medplum.startClientLogin('YOUR_CLIENT_ID', 'YOUR_CLIENT_SECRET');

    const patient: Patient = await medplum.createResource<Patient>({
      resourceType: 'Patient',
      name: [
        {
          use: 'official',
          family: 'Smith',
          given: ['Jane'],
        },
      ],
      birthDate: '1990-06-15',
      gender: 'female',
      telecom: [
        {
          system: 'email',
          value: 'jane.smith@example.com',
        },
      ],
    });

    console.log('Created patient with id:', patient.id);
    ```

    The server assigns a UUID `id` and a `meta.versionId` on creation. The returned object is the full resource as stored.
  </Step>

  <Step title="Search for resources">
    Use `searchResources` to query FHIR resources by search parameters:

    ```typescript search.ts
    import { MedplumClient } from '@medplum/core';
    import type { Patient } from '@medplum/fhirtypes';

    const medplum = new MedplumClient({ baseUrl: 'https://api.medplum.com/' });
    await medplum.startClientLogin('YOUR_CLIENT_ID', 'YOUR_CLIENT_SECRET');

    // Search patients by family name
    const patients = await medplum.searchResources<Patient>('Patient', {
      family: 'Smith',
      _sort: '-_lastUpdated',
      _count: '10',
    });

    for (const p of patients) {
      const name = p.name?.[0];
      console.log(`${name?.given?.join(' ')} ${name?.family} — id: ${p.id}`);
    }
    ```

    You can also read a single resource by its ID:

    ```typescript read-resource.ts
    import { MedplumClient } from '@medplum/core';

    const medplum = new MedplumClient({ baseUrl: 'https://api.medplum.com/' });
    await medplum.startClientLogin('YOUR_CLIENT_ID', 'YOUR_CLIENT_SECRET');

    const patient = await medplum.readResource('Patient', 'PATIENT_ID');
    console.log(patient.name);
    ```

    Or update a resource with `updateResource`:

    ```typescript update-resource.ts
    import { MedplumClient } from '@medplum/core';
    import type { Patient } from '@medplum/fhirtypes';

    const medplum = new MedplumClient({ baseUrl: 'https://api.medplum.com/' });
    await medplum.startClientLogin('YOUR_CLIENT_ID', 'YOUR_CLIENT_SECRET');

    const existing = await medplum.readResource('Patient', 'PATIENT_ID');

    const updated = await medplum.updateResource<Patient>({
      ...existing,
      birthDate: '1990-06-15',
    });

    console.log('Updated versionId:', updated.meta?.versionId);
    ```
  </Step>

</Steps>
