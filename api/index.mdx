Medplum exposes a standards-based REST API built on FHIR R4. All endpoints require HTTPS with TLS 1.2 or higher.

## Base URLs

| Environment | Base URL |
|-------------|----------|
| Medplum Cloud | `https://api.medplum.com` |
| Self-hosted | `https://your-domain.example.com` |

The FHIR base path is `/fhir/R4`. For example: `https://api.medplum.com/fhir/R4/Patient`.

## Authentication

All requests must include a Bearer token in the `Authorization` header:

```http
Authorization: Bearer <access_token>
```

Obtain an access token via the [OAuth 2.0 token endpoint](/api/oauth/index) using the authorization code flow, client credentials flow, or SMART on FHIR launch.

## Content types

| Use case | Content-Type |
|----------|-------------|
| FHIR resources | `application/fhir+json` |
| Auth and admin endpoints | `application/json` |
| OAuth token requests | `application/x-www-form-urlencoded` |

## Rate limits

Medplum applies rate limiting on all API endpoints. When a rate limit is exceeded, the server returns `429 Too Many Requests`. The response includes a `Retry-After` header indicating how many seconds to wait before retrying.

For high-volume use cases such as bulk data export, use the [Bulk FHIR API](/api/fhir/operations) which is designed for large-scale data access.

## Error format

All errors follow the FHIR [OperationOutcome](https://hl7.org/fhir/R4/operationoutcome.html) format. Each issue includes a `severity` and `code` that maps to standard [FHIR issue types](https://hl7.org/fhir/valueset-issue-type.html).

```json
{
  "resourceType": "OperationOutcome",
  "issue": [
    {
      "severity": "error",
      "code": "invalid",
      "details": {
        "text": "Invalid resource type"
      }
    }
  ]
}
```

Common HTTP status codes:

| Status | Meaning |
|--------|---------|
| `200 OK` | Request succeeded |
| `201 Created` | Resource created |
| `400 Bad Request` | Invalid request body or parameters |
| `401 Unauthorized` | Missing or invalid access token |
| `403 Forbidden` | Insufficient permissions |
| `404 Not Found` | Resource does not exist |
| `409 Conflict` | Version conflict (use `If-Match` to resolve) |
| `429 Too Many Requests` | Rate limit exceeded |
| `500 Internal Server Error` | Server error |
