## Get started in 4 steps

<Steps>
  <Step title="Register a project">
    Create a free account at [app.medplum.com](https://app.medplum.com) and register your first project. Your project is an isolated workspace for your clinical data.
  </Step>
  <Step title="Install the SDK">
    Add `@medplum/core` to your application to start interacting with the FHIR API.

    ```bash
    npm install @medplum/core @medplum/fhirtypes
    ```
  </Step>
  <Step title="Initialize the client">
    Create a `MedplumClient` with your project credentials and start reading and writing FHIR resources.

    ```typescript
    import { MedplumClient } from '@medplum/core';

    const medplum = new MedplumClient({
      baseUrl: 'https://api.medplum.com/',
      clientId: 'YOUR_CLIENT_ID',
    });

    await medplum.startClientLogin('YOUR_CLIENT_ID', 'YOUR_CLIENT_SECRET');
    ```
  </Step>
  <Step title="Create your first resource">
    Write a Patient resource to your clinical data repository.

    ```typescript
    const patient = await medplum.createResource({
      resourceType: 'Patient',
      name: [{ given: ['Jane'], family: 'Smith' }],
      birthDate: '1985-03-15',
    });

    console.log('Created patient:', patient.id);
    ```
  </Step>
</Steps>

## Platform capabilities

| Capability | Details |
|---|---|
| **FHIR Version** | R4 (4.0.1) |
| **Auth** | OAuth 2.0, SMART-on-FHIR, OpenID Connect, SAML |
| **APIs** | REST, GraphQL, WebSockets |
| **Compliance** | HIPAA, SOC 2 Type II, ONC, HITRUST |
| **Deployment** | Cloud (hosted), AWS, Azure, GCP, Docker |
| **Languages** | TypeScript/JavaScript, any HTTP client |
