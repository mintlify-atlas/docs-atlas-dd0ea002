---
title: "Client Credentials"
description: "Use the OAuth 2.0 client credentials flow to authenticate server-to-server integrations without user interaction."
---

The client credentials flow lets a backend service authenticate directly with Medplum using a client ID and secret, without any user involvement. The service receives a short-lived access token it can use to make API requests.

## When to use client credentials

Use this flow when your code runs in a trusted server environment and there is no human user logging in:

- Medplum Bots and automation scripts
- Backend integrations with EHRs, lab systems, or medical devices
- Proxy APIs that call Medplum on behalf of your own API consumers
- CI/CD pipelines that read or write FHIR resources

<Note>
For user-facing apps (web or mobile), use an identity-provider–based login flow instead. The client credentials flow is for machine identities only.
</Note>

## Step 1: Create a ClientApplication

<Steps>
  <Step title="Open the Client Applications admin page">
    Navigate to [app.medplum.com/admin/clients](https://app.medplum.com/admin/clients). You must be a project admin to access this page.
  </Step>
  <Step title="Create a new client">
    Click **New** and give your client application a name that describes its purpose (e.g., `Lab Integration Bot`).
  </Step>
  <Step title="Copy the ID and Secret">
    After saving, the page shows the **Client ID** and **Client Secret**. Store the secret securely — Medplum will not display it again.
  </Step>
</Steps>

<Warning>
Treat the client secret like a password. Store it in a secrets manager (AWS Secrets Manager, HashiCorp Vault, etc.) and never commit it to source control.
</Warning>

## Step 2: Get an access token

Send a `POST` request to the OAuth 2.0 token endpoint:

<CodeGroup>

```bash curl
curl -X POST https://api.medplum.com/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=$MY_CLIENT_ID&client_secret=$MY_CLIENT_SECRET"
```

```python python
import requests

def get_access_token(client_id: str, client_secret: str) -> str:
    response = requests.post(
        "https://api.medplum.com/oauth2/token",
        headers={"Content-Type": "application/x-www-form-urlencoded"},
        data={
            "grant_type": "client_credentials",
            "client_id": client_id,
            "client_secret": client_secret,
        },
    )
    response.raise_for_status()
    return response.json()["access_token"]
```

```typescript typescript
import { MedplumClient } from '@medplum/core';

const medplum = new MedplumClient();
await medplum.startClientLogin(MY_CLIENT_ID, MY_CLIENT_SECRET);
// The client now holds a valid access token automatically.
```

</CodeGroup>

<Note>
If you are self-hosting Medplum, replace `https://api.medplum.com` with your server's base URL (e.g., `http://localhost:8103`).
</Note>

A successful response looks like:

```json
{
  "token_type": "Bearer",
  "access_token": "<YOUR_ACCESS_TOKEN>",
  "expires_in": 3600
}
```

## Step 3: Use the token in API requests

Include the token as a Bearer credential in the `Authorization` header:

```bash
curl https://api.medplum.com/fhir/R4/Patient \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

All subsequent requests from the same `MedplumClient` instance are authenticated automatically when you use `startClientLogin()`.

## Token expiry and refresh

Access tokens issued via client credentials expire after **1 hour** (`expires_in: 3600`). The client credentials flow does not issue refresh tokens. When a token expires, repeat the token request with the client ID and secret to obtain a new one.

The `MedplumClient` SDK handles this automatically — it detects expiry and re-authenticates before making the next API call.

## Using MedplumClient

The [`startClientLogin`](https://www.medplum.com/docs/sdk/core.medplumclient.startclientlogin) method is the recommended way to use client credentials in TypeScript/JavaScript:

```typescript
import { MedplumClient } from '@medplum/core';

const medplum = new MedplumClient({
  baseUrl: 'https://api.medplum.com/',
});

// Authenticate once
await medplum.startClientLogin(
  process.env.MEDPLUM_CLIENT_ID,
  process.env.MEDPLUM_CLIENT_SECRET
);

// Make API calls — the client manages token refresh automatically
const patients = await medplum.searchResources('Patient', { _count: '10' });
```
