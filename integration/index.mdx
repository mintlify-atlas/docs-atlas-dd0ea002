Medplum supports three integration tiers: first-party integrations with common medical systems, standard medical protocols (FHIR, HL7 v2, SMART-on-FHIR), and custom integrations built with Bots and the TypeScript SDK.

## Standard protocol integrations

Medplum provides templates and playbooks for common healthcare data protocols:

| Protocol | Category | Use case |
|---|---|---|
| HL7 v2 (MLLP) | EHR / devices | ADT, ORM, ORU, SIU feeds from legacy systems |
| FHIR R4 | EHR | Read/write from FHIR-capable EHRs and payors |
| SMART on FHIR | EHR | Context-aware app launch from EHR workflows |
| FHIRcast | Radiology / EHR | Real-time clinical context synchronization |
| Bulk FHIR | EHR / payor | Export FHIR data at scale for partners |
| FHIR CMS 9115 | Payor | Provider directory for payor compliance |
| SFTP | EHR / lab | File-based data synchronization |
| OAuth2 | Identity | Plug in any OAuth2 identity provider |
| C-CDA | EHR | Import and export clinical documents |

## FHIR-based integration approach

Medplum's API is a fully conformant FHIR R4 server. Any system that can speak FHIR can integrate with Medplum directly:

- Use the [FHIR REST API](/api/fhir/overview) to create, read, update, and delete resources.
- Use the [$export operation](/api/fhir/operations) to export large datasets for analytics or partner sharing.
- Use [Subscriptions](/subscriptions/index) to trigger Bots or webhooks when resources change.
- Use the [CLI](/cli/index) to manage resources and test connectivity.

## C-CDA import and export

Medplum supports Clinical Document Architecture (C-CDA) for exchanging structured clinical documents with EHRs:

- Import C-CDA documents by posting them as a `DocumentReference` or `Binary` resource and processing them in a Bot.
- Export C-CDA by generating structured documents from FHIR resources using the Bot framework.

<Note>
  C-CDA import and export typically requires a custom Bot to handle the mapping between the C-CDA document structure and FHIR resources. Refer to the [Bots documentation](/bots) for implementation patterns.
</Note>

## Building custom integrations

Complex integrations are built by composing [Bots](/bots/index), [Subscriptions](/subscriptions/index), and the TypeScript SDK:

- **Bot webhooks** — receive inbound calls from Stripe, Acuity, Cal.com, and any webhook-capable service.
- **Bot cron jobs** — poll SFTP servers or external APIs on a schedule.
- **PDF generation** — produce CMS 1500 forms, superbills, and clinical documents.
- **FHIR operations** — build custom `$operation` endpoints backed by Bots.

See the [integration features and fixes](https://github.com/medplum/medplum/pulls?q=is%3Apr+label%3Aintegration) on GitHub for the code powering many of these integrations.
