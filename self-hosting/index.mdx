Medplum is an open-source healthcare development platform you can run in your own environment. Self-hosting gives you full control over where your data lives and how it is operated.

## Why self-host?

- **Data sovereignty** — keep patient data inside your own infrastructure and jurisdiction.
- **Compliance control** — apply your own security policies, audit logging, and access controls.
- **Cost optimization** — use existing cloud credits, reserved instances, or on-premises hardware.
- **Custom integrations** — connect directly to internal systems without traversing a third-party network.

<Note>
  Prefer a managed option? The Medplum team operates a hosted service at [app.medplum.com](https://app.medplum.com) with no infrastructure to manage. See [best practices](/self-hosting/best-practices) for operational guidance before deciding.
</Note>

## Architecture components

Every Medplum deployment requires the same core services:

| Component | Purpose |
|---|---|
| **Medplum server** | FHIR API, auth, bots, subscriptions |
| **PostgreSQL** | Primary relational database |
| **Redis** | Caching and background job queue (BullMQ) |
| **Object storage** | Binary data and file attachments |
